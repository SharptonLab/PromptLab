# Pre-Publication Verification Checklist

This checklist supports verification of work that involved large language model assistance before submission for publication, grant review, or other high-stakes contexts. Not every section applies to every project; focus on sections relevant to the specific model assistance used.

## Citations and References

Citation fabrication is among the most common failure modes of large language models. Models generate citations complete with plausible author names, journal titles, and findings, delivered with the same confidence as accurate references. Assume all model-suggested citations are fabricated until independently verified.

- Every citation has been verified to exist through PubMed, Google Scholar, or the appropriate disciplinary database
- Each cited paper has been located and its content reviewed
- Claims attributed to sources accurately reflect what those sources actually say
- Bibliographic details including authors, year, journal, volume, and pages are correct
- No citations generated by the model were accepted without independent verification

The model should never be asked to verify its own citations; it will confidently confirm fabricated references.

## Code and Computational Methods

Code that executes without errors may nonetheless produce incorrect results. Syntactically valid code can implement flawed logic, mishandle edge cases, or produce subtle errors that escape notice without systematic testing.

- All generated code has been read and understood, not merely executed
- Code has been tested with known-answer inputs where correct outputs can be independently determined
- Edge cases including empty inputs, boundary values, and missing data have been tested
- Packages and functions used are current and not deprecated
- Logic is appropriate for the actual data structure being analyzed
- Results have been spot-checked against manual calculations or alternative implementations

## Statistical Methods and Recommendations

Large language models engage with statistical concepts in ways that sound authoritative while masking serious limitations. Recommendations may seem reasonable but violate assumptions given the actual data structure, or may omit critical caveats. Treat model suggestions as brainstorming rather than authoritative guidance.

- Any model-suggested statistical approach has been verified against textbooks, methodological papers, or established guidelines
- Assumptions required by the recommended approach have been checked against the actual data
- For analyses central to the work's conclusions, a statistician or methodologist has reviewed the approach
- Alternative approaches have been considered rather than accepting the first suggestion

## Factual Claims

Confidently stated false claims are indistinguishable from accurate ones without independent verification. Models generate precise-sounding numbers and specific details that lend authority to claims regardless of accuracy.

- Specific numbers, percentages, and statistics have been traced to verifiable primary sources
- Claims about mechanisms, methods, or established findings have been verified against primary literature or authoritative references
- Historical or contextual claims have been checked for accuracy
- Any claim that would be notable if true has been independently verified

## Text and Writing Assistance

Model editing can subtly shift meaning, introduce unsupported claims, or distort emphasis in ways that escape notice when reviewing only for grammar and flow. Review text for meaning, not merely for polish.

- All text has been reviewed for meaning and accuracy, not only grammar and flow
- No unsupported claims were introduced during drafting or editing
- Technical terminology is used correctly throughout
- The final text accurately represents the data, methods, and conclusions
- Intended meaning has been preserved through any model-assisted revision

## Documentation

Adequate documentation supports reproducibility and demonstrates due diligence.

- Models used are recorded, including name and version or access date where available
- The nature of model assistance is documented appropriately per target venue requirements
- Prompts used for systematic work that shaped results are preserved and available for disclosure
- Verification procedures undertaken have been documented

## Final Verification

The principle underlying this checklist: researchers bear responsibility for every claim in their work, regardless of how that claim was generated. Model assistance does not reduce this responsibility.

- No model output was accepted without verification appropriate to its stakes
- Every statement in the work can be defended as if written independently, because authorship entails accountability for all content regardless of origin
