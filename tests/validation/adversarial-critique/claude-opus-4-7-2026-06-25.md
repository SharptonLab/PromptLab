# Adversarial Critique: Microbiome Methods Section

## MAJOR CONCERNS

### 1. "Standard protocols" is meaningless
**Quote:** *"Stool samples were collected from all participants using standard protocols."*

There is no universal "standard protocol" for stool collection in microbiome research. This phrase is a red flag for any reviewer. Critical missing details:
- Collection container/kit (OMNIgene-GUT? Sterile container? Stabilization buffer?)
- Self-collection at home vs. clinic collection
- Time from defecation to freezing (critical — microbial composition shifts within hours at room temperature)
- Aliquoting procedure
- Whether samples were homogenized before aliquoting
- Dietary restrictions, antibiotic washout periods, bowel prep status

**Fix:** Specify exact collection kit, time-to-freeze, storage/transport conditions, and any participant restrictions (antibiotics, probiotics, diet, recent colonoscopy).

### 2. "Immediately frozen" is ambiguous and likely inaccurate
**Quote:** *"Samples were immediately frozen at -80°C"*

Unless participants defecated in a lab next to a -80°C freezer, this is not literally true. Reviewers will demand the actual time-to-freeze interval and intermediate storage conditions (e.g., -20°C, dry ice, stabilization buffer). This is a known confounder in IBD microbiome studies.

### 3. Missing critical confounder controls for IBD comparison
For a healthy vs. IBD comparison (n=30/group), the Methods are silent on:
- **Disease activity/phenotype** (Crohn's vs. UC; active vs. remission; location)
- **Medication status** (5-ASA, steroids, biologics, immunomodulators all alter microbiome)
- **Antibiotic exposure** (huge confounder — typical exclusion is ≥3 months)
- **Matching criteria** for controls (age, sex, BMI, diet)
- **Recruitment site/timing** (batch effects)
- **Sample size justification / power calculation**

Without these, group differences are uninterpretable. A reviewer will reject on this alone.

### 4. Greengenes database is outdated
**Quote:** *"taxonomy was assigned using the Greengenes database"*

Greengenes has not been meaningfully updated since 2013. Current standard is SILVA (138+) or GTDB, or the newer Greengenes2. Reviewers will question taxonomic accuracy, especially for novel or recently reclassified taxa relevant to IBD (e.g., *Faecalibacterium*, *Ruminococcus* reclassifications).

**Fix:** Justify Greengenes use or switch databases; specify version.

### 5. QIIME2 "default parameters" is non-reproducible
**Quote:** *"Sequences were processed using QIIME2 with default parameters"*

QIIME2 is a pipeline, not a single tool. "Default parameters" is meaningless because:
- Which denoising? DADA2 or Deblur? (yields different ASVs)
- Truncation/trimming lengths? (Critical for V4 paired-end data)
- Quality filtering thresholds?
- Rarefaction depth for diversity metrics? (Major influence on Shannon and Bray-Curtis)
- QIIME2 version? (Behavior changes across versions)
- Classifier training method?

**Fix:** Specify version, denoising tool with parameters, truncation lengths, rarefaction depth (or alternative normalization), and classifier details.

### 6. Statistical methods are entirely unspecified
**Quote:** *"All statistical analyses were performed using appropriate methods and significance was set at p < 0.05."*

This is the most egregious sentence. "Appropriate methods" tells the reader nothing. Required:
- Test for alpha diversity comparisons (Wilcoxon? t-test? Were distributions checked?)
- Test for beta diversity (PERMANOVA? With how many permutations? Dispersion test via PERMDISP?)
- Differential abundance methods (ANCOM-BC, MaSLin2, DESeq2, ALDEx2 — all give different answers)
- **Multiple testing correction** — completely absent, critical for microbiome data
- Software/package versions
- Covariate adjustment

This sentence as written would fail review at virtually any journal.

## MODERATE CONCERNS

### 7. No mention of controls
- Negative extraction controls?
- Positive controls / mock community?
- Sequencing blanks?

Contamination is a major issue, particularly relevant in low-biomass or comparative studies. Absence of controls undermines credibility.

### 8. Single diversity metric per category
Shannon alone (alpha) and Bray-Curtis alone (beta) is limited. Reviewers often expect:
- Alpha: Shannon + Faith's PD (phylogenetic) + observed features/richness
- Beta: Bray-Curtis + UniFrac (weighted and unweighted)

This is a choice, not a requirement, but should be justified.

### 9. No sequencing depth or QC reporting
Methods should state target read depth, actual reads obtained, sample exclusion criteria for low depth, and how depth heterogeneity was handled.

### 10. Primers cited but not fully described
515F/806R has multiple variants (original Caporaso vs. updated Parada/Apprill versions). The updated versions reduce bias against *Crenarchaeota* and *SAR11*. Specify which.

## MINOR CONCERNS

- No mention of ethics approval / IRB / consent in this paragraph (may be elsewhere, but typically referenced)
- "Following manufacturer instructions" — were any modifications made? Bead-beating duration is commonly modified and matters.
- No mention of how singletons/chimeras were handled
- No data/code availability statement referenced

The vague statement that 'appropriate methods' were used is unacceptable. Without addressing these, the results cannot be evaluated or reproduced. Major revision required."

6. Add negative/positive controls
7. Specify primer variant and exact sequences
