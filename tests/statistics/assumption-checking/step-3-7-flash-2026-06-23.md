### 2. Detailed Assumption Checks
#### Assumption 1: Valid Permutation Scheme (Exchangeability)
- **Plain language**: Under the null hypothesis (no difference between groups), swapping group labels for samples should not break the natural dependence structure of your data. For example, if you have repeated measures (the same samples measured at Time 1 and 2), you cannot swap a Time 1 sample from Treatment A with a Time 2 sample from Treatment B, because those samples come from the same individual.
- **How to test**:
  1. Map your study design: Do you have repeated measures (same `SampleID` across Time points), spatial clustering (samples from the same plot/site), or other non-independent sampling?
  2. For repeated measures: Confirm each `SampleID` appears once per Time point. For spatial data: confirm clusters are nested within groups.
- **Interpretation**:
  - If you have dependence and do not account for it in permutations: assumption violated, p-values are invalid.
  - If you use a permutation scheme that respects your design (e.g., permute within `SampleID` for repeated measures): assumption met.
- **If violated**:
  Use the `strata` argument in `adonis2` to specify the grouping variable for permutation (e.g., `strata = SampleID` for repeated measures, `strata = Plot` for spatial clusters). For complex dependence (e.g., crossed random effects), use restricted permutation schemes (e.g., `how = "snedecor"` in vegan for spatial data) or alternative methods like `dbrda` with random effects.
- **Limitation**: Permutation p-values are approximate for small sample sizes; use ≥999 permutations for accuracy.

---
#### Assumption 2: Homogeneity of Multivariate Dispersions
- **Plain language**: The spread (variability) of samples within each group is roughly equal across all groups. For example, a "tight" cluster of samples in one Treatment group and a "wide" spread in another indicates unequal dispersions.
- **How to test**:
  1. Formal test: Use `betadisper()` (vegan), the multivariate analog of Levene’s test, followed by an ANOVA on the `betadisper` output to test for dispersion differences across groups.
  2. Visual diagnostic: Plot a PCoA of the Bray-Curtis distance matrix with 95% confidence ellipses for each group; overlapping ellipses of similar size indicate homogenous dispersions.
- **Interpretation**:
  - Significant `betadisper` ANOVA (p < 0.05) + visibly non-overlapping/very different sized ellipses: dispersions are unequal.
  - Non-significant `betadisper` + overlapping ellipses: dispersions are homogenous.
  - *Mitigating factor for your design*: You have equal sample sizes (10 per group), which makes PERMANOVA far more robust to moderate dispersion differences than unbalanced designs.
- **If violated**:
  1. First, check if outliers are driving the difference (see Assumption 4). Remove/ investigate data errors if present.
  2. If dispersions are still unequal:
     - For small/moderate differences (within-group dispersion ratio < 2–3) with equal n: Proceed with caution. Report both the PERMANOVA (location) and `betadisper` (dispersion) results—unequal dispersion may be a biologically meaningful result (e.g., Treatment increases community variability).
     - For large differences: Use a dispersion-robust alternative: `mrpp()` (multi-response permutation procedure) or `anosim()`, or transform your data (e.g., log-transform abundance counts before calculating Bray-Curtis) to reduce dispersion differences.
- **Limitation**: `betadisper` has low power with small sample sizes, so a non-significant result does not guarantee homogenous dispersions—always pair with visual checks.

---
#### Assumption 3: Appropriate Distance Metric
- **Plain language**: The distance you use to measure how different two samples are should match your data type and research question. For microbiome count data, Bray-Curtis is standard for relative abundance, but will be dominated by library size differences if used on raw, unnormalized counts.
- **How to test**:
  1. Check if your count data is normalized: Plot library size (total counts per sample) by group. If library sizes vary widely across groups, raw Bray-Curtis will be biased.
  2. Confirm the metric matches your question: Use Bray-Curtis for relative abundance differences, Jaccard for presence/absence only, or UniFrac if you need to account for phylogenetic relatedness.
  3. Optional: Run a Procrustes test to compare Bray-Curtis to an alternative metric (e.g., Jaccard) to confirm your metric aligns with your question.
- **Interpretation**:
  - Normalized data + metric matches your question: assumption met.
  - Raw counts used for Bray-Curtis, or metric mismatched to your question: assumption violated.
- **If violated**:
  Normalize count data before calculating distances: use total sum scaling (`decostand(otu_table, method = "total")` for relative abundance) or a variance-stabilizing transformation (VST, e.g., `vst()` from DESeq2) to account for library size differences. Switch to Jaccard/UniFrac if your question requires it.
- **Limitation**: Rarefying (subsampling to equal library size) is a common normalization approach, but it discards data and is controversial; VST or total sum scaling are preferred for most use cases.

---
#### Assumption 4: No Influential Multivariate Outliers
- **Plain language**: No single sample is so different from all other samples that it drastically changes the distance between groups.
- **How to test**:
  1. Visual: Look for samples far outside the cluster of their group on the PCoA plot.
  2. Formal: Calculate the average within-group distance for each sample; flag samples with within-group distances > 2 standard deviations above their group mean.
- **Interpretation**:
  - No flagged outliers: assumption met.
  - Flagged outliers: may be influencing results.
- **If violated**:
  1. First, check for data entry/lab errors. Remove or correct errors if found.
  2. If outliers are real biological samples: Run the analysis with and without outliers to check for result consistency. If results change, report both analyses, or use a robust PERMANOVA (add `trim = 0.1` to `adonis2` to trim 10% of the largest distances).
- **Limitation**: Outliers may be biologically meaningful (e.g., a unique responder to treatment), so do not remove them without justification.

# Load required packages
library(vegan)
library(ggplot2)
library(dplyr)

# --------------------------
# Load your data here
# otu_table: rows = samples, columns = ASVs (raw count data)
# sample_data: rows = samples, columns = SampleID, Treatment, Time

# Create interaction group variable
sample_data$Group <- interaction(sample_data$Treatment, sample_data$Time)

# --------------------------
# 1. Check permutation scheme
# --------------------------
# Check for repeated measures (same SampleID across Time points)
table(sample_data$SampleID, sample_data$Time)
# If each ID appears twice (once per Time), set strata to SampleID
strata_var <- sample_data$SampleID # Remove this line if samples are independent

# --------------------------
# 2. Check dispersion homogeneity
# --------------------------
# Normalize counts to relative abundance for Bray-Curtis
otu_rel <- decostand(otu_table, method = "total")
# Calculate Bray-Curtis distance
bc_dist <- vegdist(otu_rel, method = "bray")

# Run betadisper test
disp_res <- betadisper(bc_dist, sample_data$Group)
disp_anova <- anova(disp_res)
print(disp_anova) # p < 0.05 = unequal dispersions

# Plot PCoA with confidence ellipses
pcoa_res <- cmdscale(bc_dist, k = 2, eig = TRUE)
pcoa_df <- data.frame(PC1 = pcoa_res$points[,1], PC2 = pcoa_res$points[,2], sample_data)

ggplot(pcoa_df, aes(x = PC1, y = PC2, color = Group)) +
  geom_point(size = 2) +
  stat_ellipse(type = "norm", level = 0.95) +
  labs(title = "PCoA (Bray-Curtis) with 95% Ellipses") +
  theme_minimal()

# --------------------------
# 3. Check distance metric appropriateness
# --------------------------
# Check library size balance across groups
sample_data$lib_size <- rowSums(otu_table)
ggplot(sample_data, aes(x = Group, y = lib_size)) + geom_boxplot() + labs(title = "Library Size by Group")

# Optional: Compare Bray-Curtis to Jaccard (presence/absence) with Procrustes test
bc_jacc <- vegdist(otu_table > 0, method = "jaccard")
proc_test <- protest(cmdscale(bc_dist, k = 2), cmdscale(bc_jacc, k = 2), permutations = 999)
print(proc_test) # Non-significant = metrics give similar ordination; use Bray-Curtis if you care about abundance

# --------------------------
# 4. Check for outliers
# --------------------------
# Calculate average within-group distance per sample
sample_data$within_dist <- sapply(1:nrow(sample_data), function(i) {
  mean(bc_dist[i, sample_data$Group == sample_data$Group[i]])
})

# Flag outliers (> 2 SD above group mean)
outlier_flags <- sample_data %>%
  group_by(Group) %>%
  mutate(threshold = mean(within_dist) + 2*sd(within_dist)) %>%
  ungroup() %>%
  mutate(is_outlier = within_dist > threshold)
print(outlier_flags[outlier_flags$is_outlier, ])

# Plot outliers on PCoA
ggplot(pcoa_df, aes(x = PC1, y = PC2, color = Group, shape = outlier_flags$is_outlier)) +
  geom_point(size = 2) +
  labs(title = "PCoA with Outliers Flagged", shape = "Outlier?") +
  theme_minimal()

# --------------------------
# Run PERMANOVA with correct settings
# --------------------------
permanova_res <- adonis2(
  formula = bc_dist ~ Treatment * Time,
  data = sample_data,
  strata = strata_var, # Remove if independent samples
  permutations = 999
)
print(permanova_res)
