````
## Assumption 1 — Independent observations / valid exchangeability

### What it means

PERMANOVA uses permutations to generate a null distribution. The basic assumption is that sample labels can be permuted under the null hypothesis.

In plain language: samples must be independent unless the permutation scheme accounts for the dependence.

Examples of violations:

- Repeated measures from the same subject treated as independent.
- Multiple samples from the same plot, cage, patient, tank, litter, etc.
- Spatially or temporally autocorrelated samples.
- Batch effects perfectly aligned with treatment.

### How to check

This is mostly a design issue, not something reliably tested after the fact.

Check metadata:

```r
table(meta$Treatment, meta$Time)
table(meta$SubjectID)        # if repeated measures exist
table(meta$Batch, meta$Treatment)
table(meta$Batch, meta$Time)
```

If each row is a genuinely independent biological sample, unrestricted permutations are acceptable.

If samples are repeated from the same subject, plot, cage, etc., unrestricted permutations are invalid.

### How to interpret

- If samples are independent: proceed with standard `adonis2`.
- If there are repeated measures or blocks: use restricted permutations or a design-specific approach.
- If treatment is confounded with batch or subject, PERMANOVA cannot separate those effects.

### What to do if violated

Use restricted permutations where possible.

For simple blocking, for example:

```r
adonis2(
  bray ~ Treatment * Time,
  data = meta,
  permutations = 9999,
  strata = meta$Block
)
```

For repeated measures, be careful.

If the same subjects are measured at both time points:

- The **Time** effect can often be tested by permuting within subjects.
- The **Treatment** main effect, if treatment is assigned between subjects, cannot be tested by simply permuting within subjects.
- The **Treatment × Time** interaction may require a split-plot or repeated-measures permutation design.

In that case, consider:

- Using subject-level summaries, e.g. change in community composition from baseline to follow-up.
- Testing within-subject distance changes.
- Using methods designed for repeated-measures microbiome data.
- Consulting a statistician for a custom permutation scheme.

## Assumption 3 — Homogeneity of multivariate dispersion

### What it means

PERMANOVA tests whether group centroids differ in multivariate space. However, it can also be affected by differences in within-group spread.

In plain language: groups should have roughly similar variability in community composition.

You already noticed that one treatment group clusters more tightly than others. This is important.

### Why this matters

If groups differ in dispersion, a significant PERMANOVA result may reflect:

- True differences in group centroids,
- Differences in dispersion,
- Or both.

Equal sample sizes, as in your design, reduce but do not eliminate this problem.

### How to test

Use `betadisper`, the multivariate analogue of Levene’s test.

You should check dispersion for:

1. Treatment groups
2. Time groups
3. Treatment × Time combinations

The Treatment × Time cell-level check is especially useful for your model.

```r
library(vegan)

# Bray-Curtis distance
bray <- vegdist(otu_rel, method = "bray")

# Treatment dispersion
bd_treat <- betadisper(bray, meta$Treatment, type = "median")
permutest(bd_treat, permutations = 9999)
anova(bd_treat)
plot(bd_treat)
boxplot(bd_treat)

# Time dispersion
bd_time <- betadisper(bray, meta$Time, type = "median")
permutest(bd_time, permutations = 9999)
anova(bd_time)
plot(bd_time)
boxplot(bd_time)

# Treatment × Time cell dispersion
meta$GroupTime <- interaction(meta$Treatment, meta$Time, drop = TRUE)

bd_cell <- betadisper(bray, meta$GroupTime, type = "median")
permutest(bd_cell, permutations = 9999)
anova(bd_cell)
plot(bd_cell)
boxplot(bd_cell)

# Pairwise comparisons among group-time cells
TukeyHSD(bd_cell)
```

You can also extract distances to centroids:

```r
meta$dist_to_centroid <- bd_cell$distances

boxplot(dist_to_centroid ~ GroupTime, data = meta,
        las = 2,
        ylab = "Distance to group centroid")
```

### How to interpret

For `permutest(bd_cell)`:

- `p >= 0.05`: no strong evidence of unequal dispersion.
- `p < 0.05`: evidence that at least some groups differ in dispersion.

Important caveat: a nonsignificant test does not prove equal dispersion. With only 10 per cell, the test has limited power.

### What to do if violated

If dispersion differences are minor:

- Your equal sample sizes help.
- You may proceed, but report the dispersion results.
- Interpret PERMANOVA as detecting differences in community structure, not purely centroid differences.

If dispersion differences are strong:

- Be cautious interpreting significant PERMANOVA results.
- Report both PERMANOVA and PERMDISP.
- Inspect whether groups differ mainly in spread or in centroid location.
- Consider transformations that reduce dominance of abundant taxa.
- Consider alternative distance measures.
- Consider analyzing dispersion as a biological result.

Useful transformations:

```r

# Square-root relative abundance

```

If significance disappears after transformation, your original result may have been driven by highly abundant taxa or dispersion structure.

## Assumption 4 — Appropriate distance measure and preprocessing

### What it means

Bray-Curtis is widely used for microbiome community composition, but it is sensitive to:

- Differences in library size if raw counts are used,
- Highly abundant taxa,
- Zero inflation,
- Compositionality.

In plain language: the distance matrix should represent the biological differences you care about.

### How to check

Check library sizes:

```r

summary(meta$library_size)

        las = 2,
        ylab = "Library size")

Check sparsity:

colSums(otu > 0)
```

Check whether library size is associated with treatment or time:

```r

### Recommended preprocessing

For Bray-Curtis, usually use relative abundance or another normalization method before computing distances.

```r
otu_rel <- decostand(otu, method = "total")
bray <- vegdist(otu_rel, method = "bray")
```

If abundant taxa dominate:

```r
otu_sqrt <- sqrt(otu_rel)
bray_sqrt <- vegdist(otu_sqrt, method = "bray")

For Hellinger-transformed data:

```r
otu_hell <- decostand(otu, method = "hellinger")
hell_dist <- dist(otu_hell)

### How to interpret

If library size differs strongly by treatment or time, and you use raw counts, PERMANOVA may reflect sequencing depth rather than biological composition.

### What to do if violated

- Do not run Bray-Curtis on unnormalized raw counts unless there is a specific justification.
- Use relative abundance, variance-stabilized data, or appropriate ecological transformations.
- Perform sensitivity analyses using more than one reasonable distance/transformation.
````
