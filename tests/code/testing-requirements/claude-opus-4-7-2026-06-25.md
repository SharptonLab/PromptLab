import numpy as np

def clr_transform(counts):
    """Apply centered log-ratio transformation to count data."""
    # Add pseudocount to avoid log(0)
    counts_pseudo = counts + 0.5

    # Calculate geometric mean per sample (row)
    log_counts = np.log(counts_pseudo)

    # CLR = log(value / geometric mean)

# Testing Strategy for CLR Transform

## Key Properties to Test

The CLR transformation has well-defined mathematical properties that make testing straightforward:
1. **Each row sums to ~0** (this is the defining property)
2. **Output shape matches input shape**
3. **Scale invariance**: CLR(c·x) = CLR(x) for any positive scalar c (because the geometric mean scales with it)
4. **Known closed-form on uniform vectors**: CLR of [k,k,...,k] = [0,0,...,0]

## 4. Red Flags 🚩

| Symptom | Likely Bug |
|---|---|
| Row sums far from 0 | Wrong axis in `mean()` (using axis=0 instead of axis=1) |
| `nan` or `inf` in output | Pseudocount not applied, or applied after log |
| All values identical per row | Geometric mean computed across whole matrix, not per row |
| Output shape transposed | `keepdims=False` causing broadcast issues |
| Large counts give different CLR than scaled large counts | Geometric mean miscomputed |
| Column sums = 0 but row sums ≠ 0 | Centering along wrong axis |
