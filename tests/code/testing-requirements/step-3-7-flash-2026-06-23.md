## First: Key Properties of Correct CLR Normalization
CLR is a standard transformation for compositional microbiome count data (where counts per sample represent relative abundances of taxa). A correct implementation will:
1. Operate **per sample** (each sample is normalized independently, not across samples or features)
2. Add a small positive pseudocount to all counts to avoid `log(0)` errors (common defaults: 1, 0.5, or ½ the smallest non-zero count in the dataset)
3. Compute the **geometric mean** of pseudocount-adjusted counts for each sample (not arithmetic mean)
4. For each count in a sample, output `log( (count + pseudocount) / geometric_mean_of_sample )` (equivalent to `log(count + pseudocount) - mean(log(count + pseudocount))` per sample)
5. Produce output where the **mean of CLR values per sample is exactly ~0** (a built-in mathematical property of CLR, due to floating-point precision)

import numpy as np
def clr_normalization(counts, pseudocount=1, log_base=np.e):
    """
    Perform centered log-ratio (CLR) normalization on microbiome count data.
    
    Parameters:
    counts : array-like, shape (n_samples, n_features)
        Raw count matrix: rows = samples, columns = taxa/ASVs/features.
    pseudocount : float, default 1
        Small positive value added to all counts to avoid log(0).
    log_base : float, default np.e (natural log)
        Base of the logarithm. Common choices: np.e, 2, 10.
    
    Returns:
    clr_counts : np.ndarray, shape (n_samples, n_features)
        CLR-normalized matrix, same shape as input.
    """
    counts = np.array(counts, dtype=np.float64)
    # Step 1: Add pseudocount to avoid log(0)
    counts_adj = counts + pseudocount
    n_features = counts_adj.shape[1]
    # Step 2: Compute log of adjusted counts (handle custom log bases)
    if log_base == np.e:
        log_counts = np.log(counts_adj)
    elif log_base == 2:
        log_counts = np.log2(counts_adj)
    elif log_base == 10:
        log_counts = np.log10(counts_adj)
    else:
        # Change of base formula for arbitrary log bases
        log_counts = np.log(counts_adj) / np.log(log_base)
    # Step 3: Compute per-sample mean of log counts (this is log(geometric mean))
    log_g = np.mean(log_counts, axis=1, keepdims=True)
    # Step 4: CLR = log(counts_adj) - log(geometric mean)
    clr_counts = log_counts - log_g
    return clr_counts

## Common Pitfalls to Check In Your Code
If your code fails the tests above, check for these frequent mistakes:
1. **Global instead of per-sample normalization**: If you compute the geometric mean across all samples (instead of per sample), the per-sample CLR mean will not be zero.
2. **Missing pseudocount**: Forgetting to add a positive pseudocount will cause `log(0)` errors and `-inf` values for zero counts (ubiquitous in microbiome data).
3. **Arithmetic mean instead of geometric mean**: Using the arithmetic mean of counts (instead of the geometric mean of log counts) will break the zero-mean per-sample property.
4. **Incorrect log order**: Taking the log of counts first, then dividing by the geometric mean (instead of subtracting the log geometric mean) will produce wrong values.
5. **Transposed input**: If your code treats columns as samples instead of rows, CLR will be applied per feature instead of per sample, leading to biologically meaningless results.
6. **Inconsistent log base**: Using natural log for counts and a different base for the geometric mean will skew results.
