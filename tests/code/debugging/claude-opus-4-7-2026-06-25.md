# Debugging Your Bray-Curtis Distance Calculation

## 1. Explain the Error

The error `shapes (50,) and (100,) not aligned: 50 (dim 0) != 100 (dim 0)` means you're trying to perform an operation between two arrays of incompatible sizes. Bray-Curtis distance requires two vectors of **the same length** (since it compares element-by-element), but you're passing it a vector of length 50 and another of length 100.

## 2. Identify the Cause

The bug is on this line:
```python
distances[i,j] = braycurtis(otu_table[i], otu_table[:,j])
```

Look carefully at the indexing:
- `otu_table[i]` → row `i` → a vector of length **100** (the taxa abundances for sample `i`) ✓
- `otu_table[:,j]` → column `j` → a vector of length **50** (one taxon's abundance across all samples) ✗

You accidentally mixed row indexing (`[i]`) with column indexing (`[:,j]`). Since your matrix is **samples × taxa** (50 × 100), you want to compare two *rows* (two samples), not a row with a column.

## 3. Provide the Fix

### Minimal fix (correct indexing):
```python
for i in range(n_samples):
    for j in range(n_samples):
        # Both should be ROWS (samples), not a row vs. a column
        distances[i, j] = braycurtis(otu_table[i], otu_table[j])
```

### Better fix (more efficient + idiomatic):
```python
import numpy as np
from scipy.spatial.distance import pdist, squareform

# Load OTU table (samples x taxa)
otu_table = np.loadtxt('otu_table.csv', delimiter=',', skiprows=1)
print(f"OTU table shape: {otu_table.shape}")  # (50, 100)

# pdist computes pairwise distances between ROWS of the input matrix.
# squareform converts the condensed distance vector into a full NxN matrix.
distances = squareform(pdist(otu_table, metric='braycurtis'))

print(f"Distance matrix shape: {distances.shape}")  # (50, 50)
```

This approach:
- Avoids redundant computation (Bray-Curtis is symmetric: `d(i,j) == d(j,i)`)
- Skips the diagonal (which is always 0)
- Is implemented in optimized C code (much faster than a Python double loop)

## 4. Explain Why

Your original approach failed because of a **conceptual confusion about matrix orientation**. In a samples × taxa table:
- A **row** is a "community profile" (abundances of all taxa in one sample)
- A **column** is a "taxon profile" (abundance of one taxon across all samples)

Bray-Curtis distance compares **two community profiles**, so both inputs must be rows. By writing `otu_table[:,j]`, you were comparing sample `i`'s community profile to taxon `j`'s distribution across samples — meaningless biologically, and dimensionally invalid.

The `pdist`/`squareform` solution works because `pdist` is built specifically to compute pairwise distances between rows of a 2D array, so there's no way to accidentally mix rows and columns.

3. **Prefer high-level functions over manual loops.** `pdist`, `cdist`, and `squareform` from `scipy.spatial.distance` are designed for exactly this task and eliminate indexing bugs.

4. **Print/inspect shapes when debugging:**

5. **Use named axes** (e.g., `xarray` or `pandas` DataFrames) when working with biological data — labeled axes make orientation errors obvious because you'd write `df.loc[sample_i]` vs `df[taxon_j]` rather than ambiguous numeric indices.
