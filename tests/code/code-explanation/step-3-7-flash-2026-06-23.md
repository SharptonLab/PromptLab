First, quick context for microbiome use cases: DESeq2 is designed for count data (e.g., 16S rRNA amplicon ASV counts, shotgun metagenomic species gene counts) where values are non-negative integers representing the number of reads assigned to each taxon per sample. It models count overdispersion (common in sequencing data) to reduce false positive differential abundance calls, which is why it’s so popular for microbiome studies.
Below is the standard code structure you’ll see in almost all these papers, with explanations for each step:

#### Explanation:
- `library()` loads pre-written code packages so you can use their functions without writing everything from scratch.
- `DESeq2` is the core package for differential abundance testing.
- `tidyverse` (or subsets like `dplyr`, `ggplot2`) is often used to clean count/metadata tables before analysis, which is common in microbiome pipelines where you may need to filter low-abundance taxa or match sample IDs between tables first.

### Step 2: Import and prepare input data
```r
# Import count table: rows = taxa (ASVs/species), columns = samples, values = integer read counts
count_data <- read.csv("microbiome_counts.csv", row.names = 1, check.names = FALSE)
# Import sample metadata: rows = samples, columns = sample attributes (e.g., group, age, batch)
metadata <- read.csv("sample_metadata.csv", row.names = 1, check.names = FALSE)
# Make sure sample IDs match between the two tables!
all(colnames(count_data) %in% rownames(metadata)) # Should return TRUE if IDs align
```
#### Explanation (microbiome-specific notes):
- Count data for DESeq2 *must be raw, unnormalized integer counts* (not relative abundances like proportions or CSS-normalized values, which will break the model). Many new microbiome researchers accidentally feed normalized data into DESeq2, which is a common error.
- `row.names = 1` tells R to use the first column of the CSV (taxon names for counts, sample IDs for metadata) as row labels instead of a default numeric index.
- `check.names = FALSE` prevents R from modifying special characters in taxon/sample names (e.g., spaces, underscores, dashes in ASV IDs) which can cause ID mismatches later.
- The ID check step is critical: if your count table has samples that aren’t in the metadata (or vice versa), DESeq2 will throw an error or drop samples silently.
