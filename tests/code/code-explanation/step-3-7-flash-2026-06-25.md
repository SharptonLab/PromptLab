## 3. Key concepts (new to DESeq2/count data analysis)
- **DESeqDataSet**: A specialized R object (from the S4 class system) that acts as a single container for all inputs (counts, metadata, design) and intermediate results (normalization factors, dispersion estimates) for a DESeq2 analysis. It ensures all downstream functions use consistent inputs.
- **Design formula**: R's formula syntax (used in functions like `lm()` for linear models) that defines the statistical model. The `~` separates the outcome (OTU counts) from the predictor variables. `+` indicates additive effects (no interaction between variables; use `*` to test for interactions, e.g., `~ treatment * batch` to test if treatment effects differ by batch).
- **Size factors**: Normalization values that adjust for differences in total sequencing depth between samples. For example, if Sample A has 1M total reads and Sample B has 100k, size factors scale counts so they are comparable across samples.
- **Dispersion**: A measure of how much count data varies beyond what is expected from random sampling (overdispersion, common in count data). DESeq2 uses a negative binomial distribution, which includes a dispersion parameter to model this extra variance, instead of the simpler Poisson distribution which assumes variance equals the mean.
- **Log2 fold change (LFC)**: The log2 ratio of average normalized counts between two groups. It is the standard effect size metric for differential abundance/expression:
  - LFC = 1 → 2^1 = 2x higher abundance in the "diseased" group
  - LFC = -1 → 2^(-1) = 2x higher abundance in the "healthy" group
  - LFC = 2 → 4x higher abundance, etc.
- **Adjusted p-value (padj)**: A p-value corrected for multiple hypothesis testing (you are testing thousands of OTUs at once, so raw p-values are inflated with false positives). `padj` controls the false discovery rate (FDR): the expected proportion of false positives among all OTUs called significant. A `padj <0.05` is the standard threshold for significance.
- **Contrast**: A specification of which pairwise comparison to test when your design includes multiple groups or covariates. It is a 3-element vector: `c("variable name", "numerator group", "denominator group")`, which calculates the effect of the numerator group relative to the denominator.
- **Independent filtering**: A DESeq2 feature that automatically filters out low-count OTUs that are very unlikely to be significant, to reduce false positives. The `alpha` argument in `results()` sets the FDR threshold used for this filtering step.

## 5. Potential issues and limitations
---
1. **Incorrect one-sided hypothesis test for LFC**: As written, the `results()` call uses a one-sided default test when `lfcThreshold` is specified: it only tests for OTUs *increased* in diseased samples (H0: LFC ≤ 1 vs H1: LFC > 1), not decreased. The final `abs(LFC) >1` filter would include decreased OTUs, but their p-values are not calculated for the two-sided test, so they may be incorrectly called significant. Fix: add `altHypothesis="two-sided"` to the `results()` call to test for both up- and down-regulated OTUs with ≥2-fold change.
2. **Arbitrary low-count filter**: The `rowSums >=10` filter uses total counts across all samples, which may retain rare, noisy OTUs that are only present in 1-2 samples. A more microbiome-appropriate filter is to keep OTUs present in at least n samples (e.g., 3) with at least x counts per sample (e.g., 5).
3. **Unaccounted compositional bias**: DESeq2 is designed for RNA-seq differential expression and does not account for the compositional nature of microbiome data (total reads per sample are fixed, so changes in one OTU affect the apparent abundance of all others). This can produce biased results for some comparisons. For microbiome-specific analysis, tools like ANCOM-BC or MaAsLin2 are often preferred as they explicitly model compositional data.
4. **Batch-treatment confounding**: If all diseased samples are processed in one batch and all healthy in another, the design formula cannot separate treatment effects from batch effects, leading to biased results. Always verify that batch is not perfectly correlated with treatment group before running the analysis.
5. **Missing quality control (QC) steps**: The code skips standard pre-analysis QC, including checking for sample mix-ups/outliers via PCA/clustering, verifying samples have sufficient total read counts, and confirming batch effects are visible and correctable via the design formula.
6. **Redundant `alpha` argument**: The `alpha=0.05` in `results()` only controls DESeq2's independent filtering stringency, not the significance threshold of the final results. The explicit `res$padj <0.05` filter is still required, so the `alpha` argument is not strictly necessary here unless you want to adjust the independent filtering.
7. **Default Wald test limitations**: For small sample sizes (e.g., <10 samples per group), the default Wald test used by `DESeq()` can be underpowered. The likelihood ratio test (LRT) may be more appropriate for small n, and can be run by setting `test="LRT"` and a reduced design (e.g., `~ batch`) in the `DESeq()` call.
8. **Minor R syntax note**: The `which()` function in the final filter is unnecessary in R: `res[res$padj <0.05 & abs(res$log2FoldChange) >1, ]` works identically and is more concise.
